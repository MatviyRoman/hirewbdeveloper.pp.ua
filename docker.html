Docker & DevOps

Контейнеризація, деплой, налаштування Linux серверів

Налаштовую Docker-середовища для розробки і продакшну, конфігурую Nginx, SSL-сертифікати, CI/CD пайплайни та Linux-сервери. Кожен мій проєкт має готовий Docker Compose стек і документацію.

Docker Compose

Будую multi-container стеки: PHP-FPM + Nginx + MySQL + Redis + Queue workers. Один docker-compose up — і середовище готове.

DockerDocker ComposeNginxGitHub Actions PHP-FPMRedisMySQL 8Let's Encrypt
docker-compose.yml
version: '3.9'
services:
  app:
    build: .
    environment:
      APP_ENV: production
    volumes: [.:/var/www/html]
    depends_on: [mysql, redis]
  nginx:
    image: nginx:alpine
    ports: ["443:443","80:80"]
  mysql:
    image: mysql:8
    volumes: [db_data:/var/lib/mysql]
  redis:
    image: redis:alpine
volumes: {db_data: {}}

Ключові можливості

🐳

Docker Compose стеки

Повноцінні multi-container налаштування: PHP-FPM + Nginx + MySQL + Redis + Queue workers. Окремі файли для dev і prod середовищ.

🔄

CI/CD пайплайни

GitHub Actions для автоматичного тестування, збірки і деплою. Zero-downtime deploy через SSH. Від commit до продакшн за хвилини.

🌐

Конфігурація Nginx

Віртуальні хости, SSL через Let's Encrypt, reverse proxy, rate limiting, Gzip-стиснення, HTTP/2. Оптимально для Laravel і WordPress.

🐧

Налаштування Linux VPS

Конфігурація Ubuntu/Debian з нуля: створення користувачів, firewall (UFW), SSH hardening, cron, автоматичні бекапи.

🔒

SSL і безпека

HTTPS скрізь через Certbot з автооновленням. Security headers (HSTS, CSP), fail2ban, UFW правила — стандарт для кожного проєкту.

📦

Laravel Docker

Оптимізований Docker-образ для Laravel: OPcache, queue workers, scheduler, Horizon, Supervisor. Готовий до продакшн навантажень.

Питання та відповіді

Часті питання

Що входить у типове Docker налаштування?
+

docker-compose.yml з контейнерами: app (PHP-FPM), Nginx, MySQL 8, Redis і Queue worker. Окремі файли для dev і prod. Nginx конфіг зі SSL готовий до роботи. Dockerfile для Laravel із налаштованим OPcache і Supervisor.

Чи можете перенести існуючий проєкт у Docker?
+

Так. Контейнеризую наявні Laravel, WordPress і PrestaShop проєкти. Мігрую бази даних, налаштовую нове середовище і перевіряю що все працює без downtime.

Чи налаштовуєте CI/CD?
+

Так — GitHub Actions пайплайн: автотести на PR, збірка образу на merge в main, деплой на VPS через SSH. Повний шлях від коміту до продакшну автоматично.
